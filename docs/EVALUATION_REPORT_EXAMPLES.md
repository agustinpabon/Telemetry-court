# EvaluationReport Examples

`lib/evaluationReportV01Examples.ts` contains deterministic
`EvaluationReport v0.1` example artifacts for demos, development, tests, and
presentations.

These examples are deliberately synthetic/sanitized demo artifacts. They are
not pilot results, study results, real analyst agreement, operational telemetry
findings, or performance validation.

## Current Example

`example-evaluation-report-synthetic-overclaim-v01` exports:

- `casePackage`: the existing synthetic `CasePackage v0.1` fixture
  `pkg-synthetic-cloudtrail-iam-001`.
- `sourceReviewResults`: two deterministic synthetic `ReviewResult v0.1`
  objects with `reviewer.context: "synthetic_demo"`.
- `evaluationReport`: the `EvaluationReport v0.1` output generated by
  `aggregateReviewResultsV01(sourceReviewResults)`.

The report demonstrates validation-bench output shape and traceability:

- verdict distribution with unsupported/overclaimed and split signals;
- recommended-action distribution with rename and split recommendations;
- evidence-rating distribution with contradiction and insufficiency signals;
- selected-label distribution across two synthetic reviewers;
- disputed evidence IDs for evidence that received different ratings;
- source review IDs that point back to the ReviewResult inputs;
- compact CasePackage references that point back to the synthetic package ID,
  case ID, cluster ID, revision, and pipeline run metadata.

## Limits

Use these examples only with these constraints:

- Examples are synthetic/sanitized and safe for local demonstration.
- No live reviewer pilot is claimed.
- No real analyst agreement is claimed.
- No restricted telemetry is included.
- No operational performance claim is made.
- Examples are for demo, development, and presentation use only.

The reviewer count and agreement fields are synthetic data produced to exercise
the aggregation contract. They show how an `EvaluationReport` represents
reviewer signals, not what real reviewers concluded.

## What Not To Infer

Do not describe the examples as validation evidence, pilot validation, analyst
consensus, real-world accuracy, real Toponymy output, real ACME4 output, SOC
triage, alert triage, incident response, remediation, or raw telemetry analysis.

The examples do not add UI, live AI calls, prompt execution, external lookup,
persistence, auth, database storage, raw telemetry ingestion, Toponymy/ACME4
execution, or a reviewer-pilot workflow.
