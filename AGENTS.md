# Telemetry Court Agent Notes

- Build toward a calm evidence-review workspace, not a SOC dashboard or chatbot.
- Prefer spacious layouts, soft borders, off-white surfaces, and restrained cyan accents.
- Use only synthetic demo data. Do not introduce real telemetry, secrets, or security incident claims.
- Keep the frontend simple: App Router, TypeScript, Tailwind, no backend or API dependencies unless explicitly requested.
- Every cluster interpretation claim must map to evidence IDs so the review flow stays auditable.
- Keep the frontend data model conceptually compatible with future Toponymy-style exports, but do not integrate Toponymy, Python, or backend services yet.
